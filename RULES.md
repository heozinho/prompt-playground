# Project Rules & Memory

- **CRITICAL RULE**: Do NOT use or build "sketchpad" components/features anywhere in this project.
- **CRITICAL RULE**: After every change, commit using conventional commit style with casual 21-year-old vocab in the message body. Format: `type(scope): short description` — types are `feat`, `fix`, `chore`, `refactor`, `style`, `docs`. Body should be casual/real e.g. "fixed that bug where nothing was happening lol", "added variant cards fr fr".
